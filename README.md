# E-commerce Microservices with Go and RabbitMQ

A bilingual reference project that demonstrates a **microservices-based e-commerce architecture** built with **Go (Golang)** and **RabbitMQ** for asynchronous communication.

Um projeto bilíngue de referência que demonstra uma **arquitetura de e-commerce baseada em microsserviços** construída com **Go (Golang)** e **RabbitMQ** para comunicação assíncrona.

## Architecture | Arquitetura

![E-commerce Microservices Architecture](/asset/img/diagram2.png)

## Overview | Visão Geral

This repository acts as the **entry point** for the solution, grouping documentation and infrastructure for two Go services managed as **Git submodules**.

Este repositório funciona como o **ponto de entrada** da solução, reunindo documentação e infraestrutura para dois serviços em Go gerenciados como **Git submodules**.

### Services | Serviços

#### `microsservice-ecommerce`

**EN:** Handles the core e-commerce workflow, including order processing, persistence, and event publication.

**PT-BR:** Responsável pelo fluxo principal do e-commerce, incluindo processamento de pedidos, persistência e publicação de eventos.

**Main responsibilities | Principais responsabilidades:**
- create and process orders | criar e processar pedidos
- persist data with ORM | persistir dados com ORM
- publish notifications to RabbitMQ | publicar notificações no RabbitMQ
- apply Clean Architecture principles | aplicar princípios de Clean Architecture

#### `microsservice-ecommerce-admin`

**EN:** Consumes business events and executes administrative routines such as daily sales processing.

**PT-BR:** Consome eventos de negócio e executa rotinas administrativas, como o processamento de vendas diárias.

**Main responsibilities | Principais responsabilidades:**
- consume notifications generated by the main service | consumir notificações geradas pelo serviço principal
- register daily sales data | registrar dados de vendas diárias
- synchronize administrative information | sincronizar informações administrativas
- apply Clean Architecture principles | aplicar princípios de Clean Architecture

## Getting Started | Primeiros Passos

### Clone the repository | Clonando o repositório

Because this project uses **Git submodules**, clone it with:

Como este projeto usa **Git submodules**, faça o clone com:

```bash
git clone --recurse-submodules https://github.com/peacevan/golang-microsservice-with-rabbitmq.git
```

If you already cloned the repository without submodules, run:

Se você já clonou o repositório sem os submodules, execute:

```bash
git submodule update --init --recursive
```

## Infrastructure | Infraestrutura

## Service Configuration | Configuração dos Serviços

Each service provides a `.env.example` file with the base environment variables.

Cada serviço fornece um arquivo `.env.example` com as variáveis de ambiente base.

Typical configuration includes:

A configuração padrão inclui:
- PostgreSQL database | banco de dados PostgreSQL
- RabbitMQ host `rabbit`

Create a local environment file inside each submodule:

Crie um arquivo de ambiente local dentro de cada submodule:

## Messaging Flow | Fluxo de Mensageria

**EN:**
1. `microsservice-ecommerce` receives or processes a business action.
2. The service publishes a message/event to RabbitMQ.
3. `microsservice-ecommerce-admin` consumes the message.
4. Administrative processing is performed, such as daily sales registration.

**PT-BR:**
1. `microsservice-ecommerce` recebe ou processa uma ação de negócio.
2. O serviço publica uma mensagem/evento no RabbitMQ.
3. `microsservice-ecommerce-admin` consome a mensagem.
4. O processamento administrativo é realizado, como o registro diário de vendas.
